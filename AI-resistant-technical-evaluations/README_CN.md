## 1. 背景：一个正在被 AI 追赶的评估

2023 年 11 月，Anthropic 性能优化团队负责人 Tristan Hume 发布了一条 Twitter 招聘帖，收到超过 1000 份申请。为高效筛选候选人，团队设计了一个性能工程实操评估。

**评估设计：**
- 在模拟 TPU 架构上优化 Python 代码
- 机器特性：手动管理暂存内存、VLIW、SIMD、多核并行
- 任务：并行树遍历优化
- 提供 Perfetto 追踪文件用于指令级性能分析
- 刻意避免深度学习领域知识要求

这个评估最初非常有效——直到 Claude 变得越来越强。

## 2. AI 能力的阶梯式提升

### 性能对比（时钟周期数，越低越好）

| 执行者 | 周期数 | 上下文 |
|--------|--------|--------|
| Claude Opus 4 | 2164 | 大量测试时间计算 |
| Claude Opus 4.5 | 1790 | 随意的 2 小时会话 |
| Claude Opus 4.5 | 1579 | 2 小时 + Harness |
| Claude Sonnet 4.5 | 1548 | 大量测试时间计算 |
| Claude Opus 4.5 | 1487 | 11.5 小时 + Harness |
| Claude Opus 4.5 | 1363 | 改进的 Harness + 大量时间 |

**关键里程碑：**
- **Opus 4（2025年5月）：** 在 4 小时内超越约 90% 的人类申请者
- **Opus 4.5：** 在 2 小时内匹配顶级人类表现

Opus 4.5 的具体方法：
1. 识别初始优化瓶颈
2. 实现标准微优化
3. 发现内存带宽的变通方案
4. 达到与最佳人类结果相当的周期数

## 3. 迭代尝试

### 尝试一：数据转置问题（失败）

**设计：** 基于高效的 2D TPU 寄存器转置，需要处理 bank 冲突回避。

**结果：** Claude Opus 4.5 通过**创造性重构**解决——不是转置数据，而是转置计算。这是评估设计者未预料到的根本不同的方法。

**教训：** AI 模型能找到绕过预期难度的侧面解决方案。这就像出一道关于搬运重物的考题，学生不搬物体而是搬了目的地。

### 尝试二：约束编程谜题（成功）

**设计灵感：** Zachtronics 游戏（TIS-100、深圳 I/O）
- 最小指令集
- 优化指令数量
- 不提供可视化工具
- 候选人必须自建调试方法

**结果：** 成功抵抗了 Claude 的能力，同时有效测试了真正的优化思维。初步数据显示与候选人质量有良好相关性。

---

## 4. 核心洞见

> "原始评估有效是因为它像真实工作。替代方案有效是因为它模拟了新颖的工作。"

这里存在一个根本性张力：

| | 真实性 | 抗 AI 性 |
|---|---|---|
| **原始评估** | 高（模拟真实性能工作） | 低（AI 轻松解决） |
| **约束谜题** | 中（抽象化的优化思维） | 高（AI 难以模式匹配） |

人类专家在足够新颖的任务上仍保持优势——当既定模式不适用时，人类创造力更有价值。

## 6. 深度思考

### 对技术面试的深远影响

这篇文章的启示远超 Anthropic 一家公司。**整个科技行业的技术面试体系正在被 AI 颠覆：**

1. **LeetCode 式面试** — 已基本失效，AI 能秒解大部分算法题
2. **系统设计面试** — 仍有价值，但 AI 辅助下门槛降低
3. **实操编码评估** — 正如本文所述，难度在快速贬值
4. **约束性创新任务** — 目前最有效，但能持续多久？

### 对中国技术招聘的启示

国内大厂的技术面试（如字节跳动、阿里巴巴的编程面试）面临同样挑战。建议：

1. **从"能不能做"转向"怎么想的"** — 关注思维过程而非最终结果
2. **限制工具使用** — 或者相反，在允许 AI 辅助的情况下评估更高层次能力
3. **实时协作评估** — 与面试官的实时互动是 AI 难以模拟的
4. **持续更新评估** — 这不再是一劳永逸的工作

### 更深层的问题

当 AI 能通过所有技术评估时，我们到底在评估什么？也许未来的技术招聘需要评估的不是"技术能力"，而是"在 AI 辅助下创造价值的能力"。

## 核心要点速查

```
1. AI 能力提升速度超出预期
   → 2023 年设计的评估在 2025 年就被轻松解决

2. 创造性重构击败预期难度
   → 模型找到人类未预料的侧面解决方案

3. 约束编程谜题更抗 AI
   → 新颖的最小指令集抵抗模式匹配

4. 真实性与抗 AI 性是取舍
   → 像真实工作的评估更易被 AI 解决

5. 人类在新颖任务上仍有优势
   → 足够约束和新颖的问题仍然有利于人类创造力

6. 持续适应是必需的
   → 评估设计现在是持续的工程挑战，不是一次性工作
```
